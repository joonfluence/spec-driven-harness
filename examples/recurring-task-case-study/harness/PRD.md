## 2. 배경 및 목적

### 문제 정의
현재 태스크는 1회성으로만 생성 가능하다. 주간 회의 준비, 월간 리포트 작성처럼 반복되는 업무를 팀원이 매번 수동으로 다시 만들어야 해서, 생성을 깜빡하거나 담당자/체크리스트가 매번 조금씩 달라지는 문제가 발생한다.

### 목적
반복되는 업무 패턴을 한 번 설정하면 자동으로 태스크가 생성되도록 하여, 반복 업무의 생성 누락을 없애고 팀의 루틴 관리 부담을 줄인다.

### 목표
- 반복 태스크 설정 후 수동 개입 없이 정해진 주기로 인스턴스 자동 생성
- 반복 일정의 일시정지/재개/특정 회차 건너뛰기 지원
- 기존 캘린더 연동·알림 기능과 자연스럽게 연동

## 3. 대상 사용자

| 사용자 유형 | 설명 | 주요 니즈 |
|------------|------|----------|
| 태스크 생성자(팀 리드/매니저) | 반복 업무를 팀에 배분하는 사람 | 한 번 설정하면 잊지 않고 자동 생성되길 원함 |
| 태스크 담당자(팀원) | 반복 태스크를 실제로 수행하는 사람 | 반복 인스턴스마다 알림을 받고, 특정 회차는 건너뛸 수 있길 원함 |

비대상: 반복 일정을 팀 외부(고객, 게스트)와 공유하는 시나리오는 이번 버전에서 다루지 않는다.

## 4. 사용자 시나리오 (User Story)

- As a **팀 리드**, I want to **태스크 생성 시 반복 주기(매일/매주-요일/매월-날짜)를 설정**, so that **매번 수동으로 다시 만들지 않아도 된다**.
- As a **팀 리드**, I want to **반복 일정을 일시정지하거나 재개**, so that **휴가/프로젝트 중단 기간에는 불필요한 태스크가 생기지 않는다**.
- As a **팀원**, I want to **특정 회차만 건너뛰기**, so that **한 번의 예외 때문에 전체 반복을 해지하지 않아도 된다**.
- As a **팀 리드**, I want to **반복 일정 자체(주기·담당자)를 수정하거나 삭제**, so that **팀 구성이나 업무 패턴이 바뀌어도 대응할 수 있다**.
- As a **팀원**, I want to **반복 태스크 생성 시 기존 알림 채널로 알림을 받고 캘린더에서 확인**, so that **별도로 확인하러 다니지 않아도 된다**.

## 5. 기능 요구사항

### In-Scope (이번 버전)

| 우선순위 | 기능 | 설명 | 수용 기준 | 비고 |
|---------|------|------|----------|------|
| P0 | 반복 규칙 생성 | 매일/매주(요일 복수 선택)/매월(날짜 지정) 반복 규칙을 태스크 생성 시 설정 | 규칙 저장 후 다음 발생일이 규칙대로 정확히 계산됨 | 매월 31일처럼 없는 날짜는 6번 항목 참고 |
| P0 | 인스턴스 자동 생성 | 반복 규칙에 따라 다음 인스턴스를 미리 생성(예: 발생일 D-1) | 스케줄러가 누락 없이 인스턴스를 생성하고, 중복 생성되지 않음 | 멱등성 요구사항은 6번 참고 |
| P0 | 반복 일시정지/재개 | 반복 규칙을 정지 상태로 전환하면 인스턴스 생성이 중단되고, 재개 시 다음 발생일부터 재개 | 정지 중에는 스케줄러가 해당 규칙을 스킵함 | |
| P0 | 특정 회차 건너뛰기 | 다가오는 특정 회차 하나만 생성하지 않도록 설정 | 건너뛴 회차는 생성되지 않고, 이후 회차는 정상 생성 | |
| P0 | 반복 규칙 수정 | 주기·담당자·체크리스트 등 규칙 자체를 수정 | 수정 시점 이후 생성되는 인스턴스에만 적용, 이미 생성된 과거 인스턴스는 유지 | Out-of-Scope: 과거 인스턴스 소급 수정 |
| P1 | 반복 규칙 삭제 | 반복 규칙 삭제 시 향후 생성 중단 | 삭제 시 "향후 인스턴스만 삭제" vs "생성된 인스턴스도 삭제"를 사용자가 선택 | |
| P1 | 캘린더 연동 | 생성된 인스턴스가 기존 팀 캘린더에 자동 표시 | 인스턴스 생성/삭제 시 캘린더 이벤트도 동기화됨 | 기존 캘린더 연동 기능 재사용 |
| P1 | 알림 연동 | 인스턴스 생성 시 담당자에게 기존 알림 채널로 알림 발송 | 담당자 변경 시 알림 수신자도 함께 갱신됨 | 기존 알림 기능 재사용 |

### Out-of-Scope (다음 버전)

- 반복 일정의 팀 외부 공유(고객/게스트 초대): 권한 모델이 복잡해져 별도 검토 필요
- 과거 생성된 인스턴스의 소급 일괄 수정: 사용자가 이미 진행 중인 태스크를 건드리는 리스크가 커서 이번 버전 제외
- 반복 규칙의 A/B 조건부 로직(예: "매주 월요일이지만 공휴일이면 화요일로"): 이번 버전은 순수 주기 규칙만 지원

## 6. 비기능 요구사항

- **성능**: 스케줄러의 인스턴스 생성 배치 작업은 활성 반복 규칙 10만 건 기준 10분 이내 완료
- **정합성**: 동일 회차에 대한 인스턴스는 정확히 1개만 존재해야 함 (중복 생성 금지)
- **중복 방지(멱등성)**: 스케줄러가 재시도되거나 중복 실행되어도 동일 회차 인스턴스가 2개 이상 생성되지 않아야 함 (회차 식별자 기반 UNIQUE 제약 필요)
- **보안**: 반복 규칙 생성/수정/삭제는 해당 태스크의 **팀 리드(생성자) 권한 보유자만** 가능. 담당자(팀원)는 자신에게 배정된 인스턴스의 "건너뛰기"만 가능하고 규칙 자체는 수정 불가
- **동시성**: 반복 규칙 수정과 스케줄러의 인스턴스 생성 배치가 동시에 실행되는 경우, 배치는 **배치 시작 시점에 스냅샷한 규칙**을 기준으로 처리한다 (수정 커밋 전 상태로 처리 후, 다음 배치 회차부터 새 규칙 반영)
- **장애 대응**: 배치 실패 시 최대 3회 자동 재시도, 3회 모두 실패하면 온콜 알림 발송 + 해당 회차는 수동 생성 가능하도록 관리자 화면에 노출
- **법/규제**: 해당 없음

## 7. 기술 고려사항

- 반복 발생일 계산에 타임존을 어떻게 처리할지 -- **확인 필요** (팀 내 담당자가 여러 타임존에 걸쳐 있는 경우, "매주 월요일 09:00"의 기준 타임존)
- 스케줄러는 기존 배치 인프라(크론 기반)를 재사용한다: 별도 신규 인프라 도입 없이 기존 스케줄러에 잡을 추가하는 방향으로 확정
- 인스턴스 생성은 이벤트 발행 후 비동기 처리한다: 기존 알림/캘린더 연동이 이벤트 구독 방식이므로 동일 패턴 재사용으로 확정
